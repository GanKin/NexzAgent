**关键方法**：

1. `parse(file_path: str) -> Tuple[List[Dict], List[str]]`：
   - 使用 `openpyxl.load_workbook(file_path, data_only=True, read_only=True)` 打开文件
   - 读取第一行作为 header
   - 验证 header 包含 `代码` 和 `标的名称` 列（体系 B 标识）
   - 逐行读取（从第 2 行开始），使用 `dict(zip(headers, row_values))` 构建字典
   - 跳过 symbol 为空的行
   - 空值处理（D-08）：
     - 趋势类字段 None → `'无趋势'`
     - 数值类字段 None → `0`
     - 字符串类字段 None → `''`
   - 收集解析错误（行号 + 字段 + 原因），不中断解析
   - 返回 (records, errors)

2. `validate_header(headers: List[str]) -> bool`：
   - 检查 header 是否包含体系 B 的关键列（`代码`, `日级别趋势`）
   - 如果不包含，返回 False（后续可扩展体系 A 支持）

3. `_normalize_value(db_field: str, raw_value) -> any`：
   - 根据 db_field 名称决定类型转换
   - 数值字段转 float/int，文本字段转 str
   - 处理特殊值：趋势字段空值 → `'无趋势'`

**错误收集格式**：
```python
{"row": 5, "field": "相对强度", "error": "无法转换为数值: 'N/A'"}
```

**注意**：
- 不使用 pandas（D-05 决策）
- openpyxl 的 `data_only=True` 确保读取公式结果而非公式本身
- `read_only=True` 减少内存占用
- 确保 openpyxl 在 requirements.txt 中（添加 `openpyxl>=3.1.0`）

```python
"""市场概览 API 路由 - 数据上传、历史查询、日期列表"""

import os
import uuid
from flask import Blueprint, request, jsonify, g
from werkzeug.utils import secure_filename

from app.utils.auth import login_required, admin_required
from app.utils.logger import get_logger
from app.services.market_overview_service import MarketOverviewService
from app.services.market_overview_parser import MarketOverviewParser

logger = get_logger(__name__)

market_overview_bp = Blueprint('market_overview', __name__)

UPLOAD_DIR = os.path.join(os.path.dirname(os.path.dirname(os.path.dirname(__file__))), 'uploads', 'market_overview')
ALLOWED_EXTENSIONS = {'xlsx', 'xls'}
MAX_FILE_SIZE = 10 * 1024 * 1024  # 10MB

service = MarketOverviewService()
parser = MarketOverviewParser()
```

**端点 1: POST /api/market-overview/upload**（D-17, D-10, D-11）
- 装饰器：`@market_overview_bp.route('/upload', methods=['POST'])` + `@login_required` + `@admin_required`
- 请求：multipart/form-data，字段名 `file`
- 验证：
  - 文件存在性检查
  - 扩展名检查（.xlsx / .xls）
  - 文件大小检查（≤10MB）
- 流程：
  1. 保存文件到 `uploads/market_overview/{date}_{type}_{timestamp}_{uuid}.xlsx`（D-12）
  2. 调用 `parser.parse(file_path)` 解析 Excel
  3. 如果解析有错误（但 records 非空），继续导入但返回警告
  4. 如果解析失败（records 为空），返回 400 + 具体错误列表（D-11）
  5. 调用 `service.import_data(data_date, data_type, records, admin_id, filename, file_path)`
  6. 返回导入摘要（D-10）：
     ```json
     {
       "code": 1,
       "msg": "success",
       "data": {
         "data_date": "2026-05-18",
         "data_type": "us_market",
         "total_rows": 1000,
         "new_rows": 850,
         "updated_rows": 150,
         "warnings": ["行5: 相对强度无法转换"]
       }
     }
     ```
- 确保 `UPLOAD_DIR` 在首次上传时自动创建（`os.makedirs(UPLOAD_DIR, exist_ok=True)`）

**端点 2: GET /api/market-overview/uploads**（D-18）
- 装饰器：`@market_overview_bp.route('/uploads', methods=['GET'])` + `@login_required` + `@admin_required`
- 参数：`limit`（默认 20）, `offset`（默认 0）
- 返回上传历史列表

**端点 3: GET /api/market-overview/dates**（D-19）
- 装饰器：`@market_overview_bp.route('/dates', methods=['GET'])` + `@login_required` + `@admin_required`
- 参数：可选 `data_type` 过滤
- 返回可用日期列表（降序）

**然后在 `backend_api_python/app/routes/__init__.py` 中注册**：

- 所有异常捕获并记录到日志
